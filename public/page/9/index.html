2019篮球世界杯观感

今日中秋，未尽团圆之意，明月照常升起，天涯共此时。

2019世界杯在中国举行，有幸近距离观看了一场比赛（法国VS立陶宛，当然这场球是有争议的，最后戈贝尔的干扰球误判了），也看了中国男篮全程的发挥，以及淘汰赛阶段的一些战况。从结果上来看，中国男篮没有让国人满意，网上骂声一片，姚明主席一肩扛起责任，不过更多的国人并没有去思考为什么会出现这样的结果，只是把责任扔给李楠指导，和这届球员们而已。我现在写下这段文字也只是为了表达此刻本人的感受，来纪念本届世界杯罢了。（毫无卵用）

今晚上在CCTV5全程观看了 阿根廷对阵法国队的比赛，法国队的现场我是在南京看过的，其中有4个强点，3个NBA首发球员，10号魔术队的富尼耶，这届法国队的当家后卫，打立陶宛那场简直无所不能，还淘汰了美国队；5号巴图姆，现在在黄蜂队，之前是开拓者的先发，很好的锋线球员；27号爵士队鲁迪戈贝尔，最佳防守球员，跟立陶宛的瓦兰丘纳斯硬钢不落下风；还有一位是12号 德科洛（de colo）得分后卫，杀死立陶宛的最后一球就是他出手的。 从纸面实力来看，阿根廷队没有NBA现役球员，唯一让我熟悉的就是姚明时代火箭队的大前锋斯科拉（Scola），而今天的他表现实在是令我印象深刻，最后赢得这场半决赛，当然是阿根廷全队的努力，但是39岁的斯科拉是当之无愧的MVP（Most Valuable Player，最有价值球员）。看到他精彩的表现，我不得不想到姚明，想到今天的中国男篮。

胜利的渴望

淘汰赛阶段，每场球都是生死战，从一开场的表现来看，斯科拉连续一条龙反击上篮和三分球，奠定了这场比赛的基调，我们要赢！而回想起中国男篮对阵尼日利亚那场球，除了易建联让我看到这种气势之外，其他球员对赢球的心气都不高，虽然方硕和阿布都投进了一些三分球，但是似乎外部的压力让其他人无法正常的发挥出自己的状态，当然我觉得一方面是队员实力本身与尼日利亚球员有差距，但似乎国人让篮球承载了过多的意义，我并不是为他们洗地，不行就是不行。

篮球基本功

我们先不要谈战术，先不要骂李楠指导，虎扑上有张图，统计了这届世界杯中国男篮队员的数据，令人深刻的罚球命中率人均60~左右，ok那么我们每罚十个球都要丢四个，也就4分，波兰那场我们输了3分。

作为职业球员，我们的要求至少应该在8成的命中率吧。作为一个业余的爱好者，我能深刻感受到其中的不易，在没有对抗只是练习的时候，我的命中率也在7~8成左右，那么有了一些体力消耗，比如全场比赛，强度最高到大学院系级别，我的罚球基本在5成以下，是的，这就是一个普通人的水平吧，我个人认为，在微博喷中国男篮的键盘侠们，大多数跟我一样，或者比我还蔡，甚至不会打球，因为真正打过大学联赛和职业联赛的球员，更知道其中艰难。那么我们要对他们更宽容吗？我只是希望他们能更稳定一些，将这些篮球的基本功练的更扎实一些。

战略战术

我不懂战术，只能从浅显谈一下我的感受。防守上，今天阿根廷的防守压迫性很强，中国男篮第一场战胜科特迪瓦的大部分原因也是防守。阿根廷队各种轮转，脚下移动很快，拼命纠缠，导致富尼耶和德科洛发挥失常，法国队的进攻处于停滞状态。同样是三后卫阵容，中国男篮要认真学习一下了。进攻上，这支阿根廷队让我看到了马刺的篮球哲学（当然吉诺比利也在现场，最后与斯科拉拥抱也令人十分感动）；传球，掩护，空切，好像阿根廷人对这种集体运动有天然的敏感一样，他们还原了篮球团队运动的本质，我们当然需要球星，我们更需要团队的力量。战术上的问题，李楠指导无疑是专业的，我们这些键盘侠连个挡拆空切都打不好，就无须再进行评论了。

以上三点都只是针对篮球方面的讨论，那么真正我们需要改变的是什么呢？

篮球大环境

我们看到今年美国队派出了一波年轻阵容，最大牌的球星是原黄蜂队现凯尔特人的肯巴沃克，美国男篮又一次刷新了他们世界大赛的最差战绩：8强。是不是很讽刺？中国男篮的最好战绩就是8强，最近一次在08年奥运会的时候。其实改变篮球大环境的事情，姚明主席已经开始做了，比如将CBA做到商业化，更职业化；提高CUBA的知名度，今年夏天也有幸在浙大紫金港看了几场大学联赛的球，优酷进行直播的也不错。中国的篮球受众也是蛮多的，市场潜力也很大，真诚的希望姚明主席能把它做大做强。

其实不光是篮球，整个中国的体育大环境都不是很好，可能也是由于经济发展阶段的局限吧，不过在不远的将来，竞技体育的繁荣时代会慢慢到来。

论文闲读-1

好久没有读论文了，周五听了匹斯堡大学的yang jun教授的一场talk，主要讲GPU内存管理的一篇paper（ASPLOS-19）《A Framework for Memory Oversubscription Management in Graphics Processing Units》，正好与一作Chen Li（国防科大phd->助理研究员）在阿里实习期间有过交流，便花了半天时间研读了一番。

不想按照论文的结构来翻译了，大概按照自己的印象来记录一下自己片面的理解吧。

要解决的问题

一篇文章，主要要讲清楚你要解决的问题，从题目来看，主要是为了解决GPU显存不足而引起Page Fault等问题，导致lantency增高的问题。

那么首先解释一下显存是什么，对比CPU内存而言，显存是GPU的存储，根据CUDA的编程的模式，需要将GPU用的Data拷贝到GPU的显存中去，然后才能够让GPU使用这些数据，我们可以知道，CPU的内存可以宽展到大几百的GB，而一块GPU卡也顶多只有16~32GB的显存空间，因此Chen Li等研究这发现这个显存的大小可能成为一个影响GPU算力的一个瓶颈。

解决问题的方法

Chen li等提出了一个名为ETC的framework，来做显存的管理，实际上更重要的一点在于他对与Application的分类更为重要。

  1. 他们通过实验分析，观察到当显存不足时，不同类型的Application有对显存不同的依赖程度，根据memory coalescing的模块，来对Regular和Inregular进行区分，而同一Application中不同kernel的data share特性也很重要，通过compile-time来进行甄别；他们将这一步自动化，进而针对不同的Application进行不同的处理。
  2. 所谓ETC，那么将每个字母所代表的技术也说一下。
    • Evict
    • 这个词翻译为剔除，实际上对于显存来说，他的meta data应该与CPU 主存的单位相同，都是以页(Page)为单位，当显存满了的时候，GPU需要的数据并不在显存中，便会产生Page Fault，我个人理解为与Cache Miss 相类似。那么显存就会DMA请求一下CPU主存的数据，将它Fetch过来，那么既然我的显存空间满了，我就需要先腾出点来，然后再把On demand的数据放进去吧，所以他们提出一种对显存数据提前剔除的一个策略Proactive Eviction；简称为E。
    • Throttling
    • 我暂时将它翻译为 限制 吧。怎么讲呢，产生page fault的原因就是说GPU Core算的太快，数据还没准备好呢，然后就去取数据，没取到的话，就出现错误了。那么怎么让减少page fault呢，我算的慢点行吗，GPU Core算的慢一些，这样是不是会好一些呢。权衡一下page fault牺牲的时间，和我将GPU Core放慢(关掉几个)算力是下降了一点，是不是总体的性能会好一些呢。因此，他们提出一种叫做Memory-aware Throttling的解决方法，具体算法就不细说了。
    • Compression
    • 压缩，这个方法呢，实际上是从CPU内存管理来的，我对CPU内存管理并不熟悉，他们借鉴了LCP （is a low-latency main memory compression framework that has been shown to effectively increase memory capacity in a CPU system.）这种压缩方式，发现对GPU的显存也很有效，然后就用上了。。
我的感受
  1. 这篇文章对我的感觉是什么样的呢，工作量很足，使用的方法呢不能说是令人惊艳，但是也是有理有据，令人信服的。
  2. 实际上显存不足的问题并非不重要，在实际工业届的话，想PCIE的带宽显得更为重要，而随之GPU等加速的算力越来越强，对CPU的挑战也很大。
  3. 对于科研本身，我似乎已经渐行渐远了，但是对于体系结构研究问题、解决问题的方法也有了一些更深的理解
    • 如何打满流水线？PingPong
    • 如何减少算力与访存之间的差距？怎么trade off？
      • 降低Cache Miss？
      • 降低Page Fault？
      • 降低算力，不要把PCIe搞的太狠？
  4. 当然好的合作者十分重要。
    • Onur Mutlu这位大神，体系结构名人堂人物。
    • jun yang 和 youtao zhang 两位教授（伉俪）
    • Rossbach这位还是操作系统领域的大神
  5. 良好而完备的工程实验环境。在做research的时候，搭实验环境往往十分耗费精力，像这种积累深厚的实验室，实现idea的速度也是极快的。
  6. 那么如何将体系结构中的方法论，应用到实际的工作中，还需仔细思考一下。
